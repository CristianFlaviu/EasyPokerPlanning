# UX Review: New Flow Proposals

## Goal

Explore whether the app would benefit from revised user flows, while staying within the v1 domain model.

## Read First

- `docs/ai-ux-review/review-guide.md`
- `docs/domain-model.md`
- `docs/progress.md`

## Tooling

Use the Browser plugin/skill to inspect current flows before proposing changes. Do not change code.

## Review Scope

Consider improvements to:

- First-time room creation.
- Shared-link join.
- Moderator setup and control flow.
- Voting and reveal flow.
- End-round and final-estimate flow.
- History discovery after a session.
- Reconnect/reload recovery.

## Constraints

Stay inside v1 scope:

- No accounts.
- No custom card decks.
- No ticket integration.
- No ownership transfer.
- No time-boxed voting.

## Output Required

Propose 2-3 alternative flows. For each:

- Describe the flow.
- Explain the user problem it solves.
- List UI changes needed.
- List backend/API changes, if any.
- Identify implementation risk.
- Recommend whether to implement now, later, or not at all.
